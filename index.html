23年鲲鹏杯赛前逆向测试题babyarm

发表于 2023-04-05 阅读次数： 本文字数： 1.3k 阅读时长 ≈ 9 分钟

打开asm文件得到以下汇编代码：
DATAS SEGMENT
STRING DB 32 DUP(?)
SRE DB 35, 110, 33, 123, 33, 103, 80, 36, 97, 80, 39, 86, 32, 106, 80, 32, 84, 106, 39, 36, 105, 80, 34, 87, 32, 110, 36, 81, 59, 97, 87, 44
RIGHT DB ‘RIGHT’
ERROR DB ‘ERROR’
DATAS ENDS

CODES SEGMENT
ASSUHE CS:CODES
START:
ASSUHE ES:STR DS:STRING
MOV AX,STRING
MOV DS,AX
MOV AX,SRE
MOV ES,AX
XOR SI,SI
XOR AX,AX
CMP:
CMP SI,1CH
JZ SHORT R1
SOUL:
MOV AX,DS:[SI]
MOV BL,9H
SUB AX,BL
XOR AX,11
INC SI
CMP AX,ES:[SI-1]
JZ CMP
R1:
LEA DX,RIGHT
MOV AH,9
INT 21H
JMP P
R2:

INT 21H
JMP P
P:
XOR AX,AX

END START

看起来不难，直接问chatgpt这段代码有什么用，它的回答：

这段汇编代码是一个简单的字符串匹配程序，具体功能如下：

1.在DATAS段中，定义了两个字符串：一个长度为32的字符串变量STRING和一个包含一系列数字的字符串变量SRE。
2.定义了两个字符数组变量RIGHT和ERROR，用于在匹配成功或失败时输出相应的提示信息。
3.在CODES段中，程序入口为START。
4.首先将DS寄存器设置为STRING变量的段地址，ES寄存器设置为SRE变量的段地址，以便程序能够访问这些变量。
5.将SI和AX寄存器都清零，用于后面的循环比较操作。
6.然后开始循环比较，比较方式是将STRING变量中的每个字符减去9，再异或11，得到一个新的字符值，然后与SRE变量中对应位置上的字符值进行比较。
7.如果比较结果相等，则继续比较下一个字符；如果不相等，则跳转到R2，输出匹配失败的提示信息。
8.如果比较完整个字符串后都匹配成功，则跳转到R1，输出匹配成功的提示信息。
9.程序结束，控制权返回操作系统。

显然就是把flag加密了，flag就是string，让chatgpt写一个还原出string的python程序，注意告诉它还原方法并指正它的错误：

告诉gpt:写一个python程序还原出string的内容，sre的数值使用汇编代码里的，异或11的逆操作也是异或11，sre用十进制表示

它回答：
sre = [35, 110, 33, 123, 33, 103, 80, 36, 97, 80, 39, 86, 32, 106, 80, 32, 84, 106, 39, 36, 105, 80, 34, 87, 32, 110, 36, 81, 59, 97, 87, 44]
string = “”

for i in range(len(sre)):
ch = (sre[i] ^ 11) + 9
string += chr(ch)

print(string)
